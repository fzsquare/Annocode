# Acceptance Handoff

Write handoffs/05-acceptance-to-orchestrator.md:

    Requirement-ID: feature-example
    From-Role: acceptance
    To-Role: orchestrator
    Attempt: A1
    Verdict: PASS | FAIL | BLOCKED

    ## Original requirement coverage
    ## Acceptance matrix
    ## Evidence used
    ## Non-goal and compatibility check
    ## Residual risks
    ## Required rework or user decision
    ## Final rationale

A PASS requires evidence for every mandatory requirement. Keep the rationale concise and auditable.
