# 变更规划检查表

## 代码勘察
- 入口：路由、命令、事件、UI、定时任务。
- 核心：服务、领域对象、状态机、算法。
- 边界：数据库、缓存、队列、外部 API、文件系统。
- 横切：鉴权、审计、日志、指标、错误处理、国际化。
- 交付：配置、迁移、部署、回滚、文档。
- 证明：单测、集成测试、端到端测试、测试夹具。

## 任务契约完整性
- Task ID 和目标。
- 依赖和 Dependency Snapshot。
- Read set / Write set。
- 要修改的文件、符号和预期行为。
- 接口输入、输出、错误和兼容性。
- 验收标准 AC 映射。
- 测试命令和完成定义。
- 消息文件路径。

## 并行判定
两个任务仅在以下条件同时满足时并行：
1. 写入文件集合不相交；
2. 不依赖对方尚未稳定的接口；
3. 测试或迁移不会争用不可隔离的共享状态；
4. 任一任务失败不会让另一个任务产生不可逆修改。
